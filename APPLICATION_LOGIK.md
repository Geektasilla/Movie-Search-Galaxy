# Архитектура и Логика консольного приложения "MovieSearch"

**Название:** MovieSearch
**Формат:** консольное приложение, должно запускаться на любой ОС (Windows/macOS/Linux).
**Цель:** Предоставить клиенту Userfriendly интерактивное консольное приложение для поиска фильмов по базе данных sakila.

## 1. Структура проекта (Модули)
Проект разделен на логические модули в соответствии с принципом единственной ответственности (Single Responsibility)и для предотвращения дублирования кода:

*   **`main.py`** — Точка входа в приложение. Отвечает за запуск, инициализацию, бесконечный цикл взаимодействия с пользователем (меню), роутинг команд и обработку глобальных команд (`end`, `back`, `help`).
*   **`mysql_connector.py`** — Слой доступа к данным (DAO) для MySQL. Содержит функции:
    *   Получение списка всех жанров.
    *   Получение минимального и максимального года выпуска.
    *   Поиск фильмов по названию (с `LIMIT` и `OFFSET`).
    *   Поиск фильмов по жанру и диапазону годов (с `LIMIT` и `OFFSET`).
    *   Поиск фильмов по жанру (с `LIMIT` и `OFFSET`).
    *   Поиск фильмов по году (с `LIMIT` и `OFFSET`).
*   **`log_writer.py`** — Слой записи данных в MongoDB. Принимает параметры поиска и количество найденных результатов, формирует документ и сохраняет его в коллекцию.
*   **`log_stats.py`** — Слой аналитики данных из MongoDB. Извлекает топ-5 популярных запросов и список последних запросов пользователя.
*   **`formatter.py`** — Модуль визуального отображения (View). Содержит функции для форматированного вывода текста в консоль (таблицы с фильмами, списки жанров, приветствие, прощание, меню и меню помощи).

## 2. Глобальные правила интерфейса
Для обеспечения "User Friendly" опыта на протяжении всей работы приложения действуют следующие глобальные команды:
*   **`end`** — прерывание текущего действия, закрытие соединений с БД, вывод красивого прощания и выход из программы. - доступно всегда
*   **`back`** — отмена текущего ввода и возврат в главное меню.
*   **`help`** — вызов справки (список доступных действий). - доступна всегда

## 3. Схема потока выполнения (Application Flow) и Валидация
### 3.1. Правила обработки пользовательского ввода (Универсальный обработчик)
Чтобы исключить падение программы при вводе некорректных данных, вводится единый механизм валидации ввода для всех меню:

*   **Регистронезависимость и пробелы**: Любой ввод пользователя автоматически очищается от лишних пробелов по краям и переводится в нижний регистр (метод `.strip().lower()`). Команды END, End, end будут распознаны корректно.
*   **Пустой ввод**: Если пользователь нажал Enter ничего не введя (или ввел только пробелы), выбрасывается предупреждение: "Ошибка: Ввод не может быть пустым. Пожалуйста, повторите".
*   **Неверный тип данных**: При ожидании номера меню (1, 2, 3, 4, 5) ввод букв, спецсимволов или текста на другом языке обрабатывается блоком `try...except` или `if in [...]`. Выводится сообщение: "Ошибка: Некорректный ввод. Пожалуйста, введите номер из предложенного меню".
*   **Неверные данные фильтров**: При вводе годов проверяется, что это числа (`isdigit()`), и что Год От <= Год До. При ошибке запрашивается повторный ввод без вылета в главное меню.

## 4. Алгоритмы работы ключевых функций

### 4.1. Пагинация результатов поиска (Pagination)
Чтобы не перегружать консоль, результаты возвращаются порциями:
1.  Выполняется SQL-запрос с `LIMIT 10 OFFSET 0`.
2.  Если возвращено 10 результатов, пользователю задается вопрос: `"Показать следующие 10 результатов? (Enter - да / back - в меню / end - выход)"`.
3.  При согласии `OFFSET` увеличивается на 10 и запрос повторяется.
4.  Процесс повторяется, пока база данных не вернет меньше 10 записей, что означает конец списка.

### 4.2. Логирование запросов
Каждый раз, когда пользователь инициирует поиск и MySQL возвращает результат (даже пустой), вызывается функция логирования `log_writer.py`. 
Создается документ JSON следующего формата:
```json
{
  "timestamp": "2025-05-01T15:34:00",
  "search_type": "keyword", 
  "params": {
    "keyword": "matrix"
  },
  "results_count": 3
}
```
*В случае поиска по жанру и годам, поле `search_type` будет равно `genre_year`, а `params` будет содержать `genre_id`, `min_year`, `max_year`.*

### 4.3. Обработка ошибок
*   Потеря соединения с MySQL/MongoDB должна перехватываться конструкцией `try...except`. Приложение должно сообщать `"Проблема с подключением к базе данных"` и не «падать» с длинным Python Traceback.
*   Неверный ввод (буквы вместо цифр при вводе года, выбор несуществующего пункта меню) перехватывается с просьбой повторить ввод.
