## Шаг 5: Сборка логики и консольного меню (`main.py`)
Здесь будет бесконечный цикл `while True`. Старайтесь сделать `main.py` чистым контроллером.

1. Оберните весь блок кода в `try...except`, чтобы приложение не вылетало с ошибкой, а писало "Произошла ошибка базы данных".
2. **Главное меню**:
   - `1. Поиск по ключевому слову или нескольким словам (полному названию)`
   - `2. Поиск по жанру и годам`
   - `3. Поиск по жанру`
   - `4. Поиск по году`
   - `5. Топ-5 популярных запросов`
   - `0. Выход`
3. **Логика поиска по слову**:
   - Запрашиваем слово.
   - `offset = 0`.
   - В цикле: вызываем метод поиска из MySQL. Если фильмов 0, пишем "Ничего не найдено", прерываем.
   - Выводим 10 фильмов с помощью `formatter.py`.
   - Сохраняем лог в MongoDB (только при первом поиске, чтобы не дублировать логи при пагинации).
   - Спрашиваем: "Нажмите Enter для следующих 10 или 'q' для выхода". Если Enter -> `offset += 10`.
4. **Логика поиска по жанру**:
   - Сначала запрашиваем и выводим минимальный и максимальный год (вызов MySQL + `print`).
   - Запрашиваем и выводим все жанры.
   - Просим пользователя выбрать жанр и годы.
   - Далее логика пагинации (`offset`) абсолютно такая же, как в поиске по слову.

## Советы из лучших практик
- **DRY (Don't Repeat Yourself)**: Логику с циклом пагинации "Показать еще 10?" можно вынести в отдельную функцию внутри `main.py`, чтобы не писать её дважды для поиска по слову и по жанрам.
- **Освобождение ресурсов**: Обязательно используйте конструкцию `with MySQLConnection(...) as db:` при каждом обращении к базе данных.
